A Modified Shortest Path Problem

Why this is interesting

Alternation constraints show up in routing with service classes (e.g., white→green→red lanes), workflow pipelines where resources must be used in order, and games where moves alternate across types. Rather than invent a new algorithm, we can reduce the problem to a classic shortest path on a slightly larger graph. This keeps the solution simple, correct, and efficient.

Problem

We are given a directed graph G=(V,E) with non‑negative weights w(e) and a color on each vertex c(v) ∈ {W, G, R}. We want a shortest path that starts on white and strictly alternates colors W→G→R→W→… at each step. Let δ_BG(s, u) denote the constrained shortest distance from s to u under this rule.

Solution sketch

Construct a layered graph G' with three copies of V: V_W, V_G, V_R. A node v appears as v^W, v^G, v^R, representing “arrive with next required color W/G/R”. For every edge (u→v) in G and for every layer X, add an edge from u^X → v^Y if c(v) = Y and Y is the next color after X in the cycle W→G→R→W. Give it the same weight w(u,v). Now any path in G' corresponds to an alternating‑color path in G.

Initialize at s^W (we must visit a white node first). Run any single‑source shortest path algorithm appropriate for weights—Dijkstra if non‑negative (typical), otherwise Bellman‑Ford. The constrained distance is δ_BG(s, u) = min{ dist(u^W), dist(u^G), dist(u^R) }.

Correctness

Claim. Paths from s^W to any u^* in G' correspond 1‑to‑1 with paths in G that start on white and alternate colors. The layer index enforces which color must be visited next, so every step is legal, and every legal alternating step has a corresponding edge in G'. Edge weights are preserved, so path weights match.

Complexity

G' has 3|V| nodes and at most 3|E| edges. Complexity matches your SSSP choice on G' (e.g., Dijkstra: O((|V|+|E|) log |V|) up to constants; Bellman‑Ford: O(|V||E|)).

Takeaways

  • Modeling trick: encode constraints via state (layers), not custom algorithms.
  • Reusability: leverages standard SSSP routines for constrained problems.
  • Extensibility: alternation over k colors → k layers; other constraints fit similar automata‑style constructions.
