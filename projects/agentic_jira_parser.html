Agentic Jira Ticket Parser & PR Creator

A workflow agent that consumes Jira tickets, generates an implementation plan, edits code, and opens GitHub pull requests automatically with structured reasoning and guardrails.

Overview

The agent turns Jira tickets into ready‑to‑review PRs. It parses the ticket, proposes a change plan, edits code, runs checks, and opens a PR with a clear summary and checklist. The system is designed for predictability and operator control.

Workflow


POST /webhooks/jira
  → validate(ticket)
  → plan = planner(ticket)              // break down into atomic tasks
  → changes = editor.apply(plan)        // file edits, codegen, refactors
  → ci = run_checks()                   // linters/tests/static analysis
  → pr = github.open_pr(changes, plan.summary, checklist)

Ingestion


// Example Jira payload (trimmed)
{
  "issue": {
    "key": "PROJ-123",
    "fields": {
      "summary": "Add dark mode to settings",
      "description": "As a user... Acceptance: toggle; persists; unit tests"
    }
  }
}

Payloads are validated against a JSON schema. Rich text is normalized into a compact spec for the planner.

Planner

  • Extracts intent, constraints, acceptance tests.
  • Generates a deterministic plan (task tree) with file‑level actions.
  • Asks clarifying questions if information is missing (optional human‑in‑the‑loop).

// Plan snippet
- create settings/ThemeToggle.tsx
- wire into pages/Settings.tsx
- add useTheme hook & localStorage persistence
- unit tests for toggle + persistence

Editor

  • Applies edits with structured diffs and idempotent transforms.
  • Runs pre-commit hooks, typechecks, and tests; failures feed back into the plan.
  • Scopes changes to a feature branch with branch protection compliance.

Version control & PR


# Branch naming
feat/PROJ-123-dark-mode-toggle

# PR body (generated)
Title: PROJ-123: Add dark mode toggle

Summary
- Adds ThemeToggle component
- Persists preference
- Test coverage for reducer + UI

Checklist
- [x] Unit tests pass
- [x] Lint & typecheck clean
- [x] Docs updated

Safety and guardrails

  • Permissions: repo allowlist, file‑path restrictions, protected directories.
  • Dry‑run mode: generates a draft PR without merging permissions.
  • Rollback: branch deletion on failure; no force‑push; audit log preserved.
  • Rate limits: per‑repo and per‑ticket quotas to prevent spam.

Results

  • Reduced PR preparation time from ~40 min to ~8–12 min (human review included).
  • Improved PR consistency: standardized titles, bodies, and checklists.
  • Lowered review friction by keeping diffs small and well‑scoped.

Quickstart

  -H 'Content-Type: application/json' \
